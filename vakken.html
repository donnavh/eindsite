Vakken

WAFS

In dit eerste vak, Web Apps From Scratch (WAFS), heb ik voor mezelf weer even opnieuw moeten leren hoe ik met code om moest gaan. Ik heb veel geleerd over de basis van HTML, CSS en JavaScript. Dit was voor mij een goede opfrisser omdat ik al een hele tijd geen code meer had geschreven en weer even goed moest inkomen. Het eerste vak in deze minor begon dus meteen met mezelf uitdagen en een nieuwe techniek uitproberen. En hier begon ik er al een beetje achter te komen dat het toch best leuk was om met code te zijn als je het eenmaal onder de knie hebt. Tuurlijk was het nieuw, maar ik merkte dat dingen al sneller gingen zoals ik het wilde. Vergeleken met voorheen.

CSS

CSS draaide vooral om, inderdaad CSS. Hier hebben we een heel project gemaakt met bijna alleen CSS. Wederom had ik zoiets nog nooit gedaan, en merkte dat ik in het begin het dan toch lastig vond om te beginnen. Vooral omdat ik simpelweg niet goed wist waar ik moest beginnen. Toch ging het na een tijde best redelijk toen ik een beetje doorhad wat ik moest doen. Hier heb ik wel geleered hoeveel er mogelijk is met CSS, als je maar durft te proberen en als je weet wat je moet gebruiken.

BT

Bij Browser Technologies heb ik geleerd hoe belangrijk het is om nette code te schrijven, maar ook code die overal werkt. We hebben hier een belastingformulier nagemaakt in de stijl van de NS dat op elk apparaat zou moeten werken. Hierbij merk je dus hoe belangrijk sommige elementen zijn en dat niet alles toegankelijk is om overal te gebruiken. Toen ik mijn herkansing ging maken voor dit vak en terug ging kijken naar mijn eerste code, zag ik dat deze slordig was en het daarom misschien ook minder goed zou werken dan ik zou willen. Maar later, toen ik dit had verbeterd, was de website ook meteen sterker in mijn ogen.

HCD

Bij Human Centered Design hebben we geleerd hoe we toegankelijk kunnen werken. Hier was het niet toegankelijk voor iedereen, maar voor een specifiek persoon. Juist om te leren begrijpen hoe moeilijk het internet kan zijn voor mensen die een bepaalde beperking hebben. Hoewel ik dit een lastig project vond om te realiseren, omdat het zo anders was dan wat ik ooit heb gedaan, heb ik hier wel veel van geleerd. Ik heb geleerd dat je niet altijd moet denken vanuit jezelf, maar ook vanuit de gebruiker. Daarbij dus vooral de gebruikers die sommige dingen anders ervaren dan mensen zonder beperking.

API

Toen we begonnen met het vak API, had ik hier nog nooit van gehoord en ik had nog nooit iets vergelijkbaars gezien. Ik denk dat ik over dit vak ook het meest sceptisch was, omdat ik niet goed wist wat ik ervan moest verwachten. Bij de andere vakken had ik een beter idee en kon ik me er iets bij voorstellen. Maar dit was voor mij echt een compleet nieuw iets. Toen we begonnen met deze code te schrijven, was het voor mij denk ik ook de eerste twee weken erg wazig en kwam ik er niet helemaal uit. Ik ging vragen stellen aan klasgenoten en docenten, maar er kwam geen logica achter. Toen dacht ik: laat ik maar een functie gaan schrijven om het misschien iets beter te begrijpen. Waarna ik langzaam een beetje verbanden begon te zien hierin. Waar het op neerkwam in mijn hoofd, was dat je ergens informatie vandaan haalt, vertelt hoe je dit wil terugkrijgen en waar dit naartoe gestuurd moet worden. Het is natuurlijk iets meer, maar in essentie komt het hierop neer. En uiteindelijk viel dit vak dus eigenlijk ook wel mee, terwijl ik hier erg bang voor was.
